# Data Directory Structure

This directory contains all data files for the Global Risk Interconnection Platform.

## 📁 Directory Organization

### `raw/`
**Purpose**: Original, unprocessed data from external sources  
**Status**: ⚠️ Ignored by Git (large files)

Place here:
- Original CSV downloads from data providers
- Raw JSON/XML exports
- Original Excel files
- Source documents

Example:
```
data/raw/climate/rainfall_2020.csv
data/raw/economy/gdp_data.xlsx
```

---

### `processed/`
**Purpose**: Cleaned, transformed, and feature-engineered data  
**Status**: ⚠️ Ignored by Git (regeneratable)

Subdirectories:
- `climate/` - Climate risk data and features
- `economy/` - Economic indicators
- `geopolitics/` - Political risk data
- `infrastructure/` - Infrastructure metrics
- `migration/` - Migration patterns
- `social/` - Social indicators
- `trade/` - Trade data

Example:
```
data/processed/climate/climate_features.csv
data/processed/climate/climate_risk_district.csv
data/processed/interconnection/interconnected_risk.csv
```

**Note**: These files are generated by pipelines and can be regenerated.

---

### `external/`
**Purpose**: Third-party datasets and reference data  
**Status**: ⚠️ Ignored by Git

Place here:
- Pre-trained embeddings
- External lookup tables
- Reference datasets from other projects

---

## 🔄 Data Flow

```
raw/ → [Processing Pipeline] → processed/ → [ML Model] → predictions
```

1. Download raw data → `data/raw/`
2. Run processing scripts → generates `data/processed/`
3. Train models → generates `models/trained/`
4. Make predictions → saved to `data/processed/`

---

## 📦 What's in Version Control

✅ **Included**:
- This README
- `.gitkeep` files (preserve structure)
- Small sample files (< 1MB)
- Metadata/documentation files

❌ **Excluded** (in .gitignore):
- Large CSV files (> 10MB)
- Raw data dumps
- Generated processed data
- Model binaries (.pkl, .h5, etc.)
- Database files

## 🚀 Getting Started

### For New Contributors:

1. **Clone the repository**
   ```bash
   git clone <repo-url>
   cd global-risk-interconnection-platform
   ```

2. **Directory structure is already created** via `.gitkeep` files

3. **Add your raw data** to appropriate subfolders in `data/raw/`

4. **Run processing pipelines** to generate processed data
   ```bash
   python pipeline/processing/climate_features.py
   python pipeline/processing/climate_model.py
   ```

5. **Processed files will be auto-generated** in `data/processed/`

---

## 🔒 Sensitive Data

**NEVER commit**:
- API keys
- Database credentials
- Personal identifying information (PII)
- Proprietary datasets without permission
- Large proprietary databases

Use environment variables or `.env` files for credentials.

## 🛠️ Maintenance

### To regenerate all processed data:
```bash
# Run all processing pipelines
python pipeline/processing/climate_features.py
python pipeline/processing/climate_model.py
python backend/app/services/interconnection_engine.py
```

### To clean up (if needed):
```bash
# Remove all generated data (KEEP RAW DATA!)
rm data/processed/**/*.csv
rm models/trained/*.pkl
```

---

## 📝 Best Practices

1. **Keep raw data immutable** - Never modify files in `raw/`
2. **Document transformations** - Update processing scripts when data changes
3. **Version control scripts, not data** - Store code that generates data
4. **Use meaningful filenames** - Include date/source in filename
5. **Compress large files** - Use .gz or .zip for archival
6. **Sample large datasets** - Create small samples for testing
